# ChemMind —— 面向化学科学的 AI Agent 系统

> 让每一篇文献、每一个实验、每一次模拟，都变得可检索、可推理、可复用。

---

## 项目概述

**ChemMind** 是一个专为化学化工场景设计的 **AI Agent 系统**，旨在通过大语言模型（LLM）构建面向化学科研全流程的智能中枢。系统将 LLM 的长上下文能力、检索增强生成（RAG）、工具调用（Function Calling）与化学领域知识图谱深度融合，覆盖从文献调研 → 实验设计 → 计算模拟分析 → 报告产出的完整科研闭环。

不同于通用型 AI 助手，ChemMind 的差异化定位在于：**为化学科研场景提供深度领域适配**——理解化学术语的精确含义、掌握反应机理的逻辑规则、具备计算化学后处理分析能力。

### 核心技术点

| 技术 | 实现方式 | 说明 |
|------|---------|------|
| **RAG** | ChromaDB + Sentence Embedding | 文献与知识库的语义检索底座 |
| **Agent** | ReAct 模式任务规划-执行循环 | 多步推理与工具调用的编排框架 |
| **Function Calling** | 结构化的工具描述与参数绑定 | 调用外部计算引擎（RDKit等） |
| **长上下文** | 滑动窗口 + 语义压缩策略 | 利用MiMo 1M token处理超长输入 |
| **结构化输出** | JSON Schema 约束 + 多轮校验 | 保证输出格式的一致性与可靠性 |
| **多模态** | 图表/谱图/分子结构的解析管线 | 处理PDF中的图表与公式 |

## 核心功能详解

### 1. ChemRAG —— 化学文献智能引擎

**技术实现**：多层级 RAG 架构

```
用户问题
    │
    ▼
┌──────────────────────┐
│  Query Rewriting     │  问题重新表述与消歧
│  + 意图分类          │  （"合成布洛芬"→"ibuprofen synthesis routes"）
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  多路并行检索         │
│  ┌─────┐ ┌─────┐    │  教材库 / 文献库 / 反应库 / 网络
│  │     │ │     │    │  四路并行，各取Top-5
│  └─────┘ └─────┘    │
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  Reranking + 去重     │  Cohere Rerank 3 + 语义去重
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  Generation          │  LLM综合生成 + 引用标注
└──────────────────────┘
```

**关键特性**：
- 支持多语种文献（中/英/日/德化学文献）
- 输出含引用溯源（"根据JACS 2024, 146, 12345..."）
- 自动提取化合物名称、CAS号、产率、表征数据并格式化

### 2. SynthFlow —— 合成路线推理引擎

**核心逻辑**：基于规则 + LLM 推理的混合系统

```
目标分子输入
    │
    ▼
┌──────────────────────┐
│  Retrosynthetic      │  基于RDKit的逆合成分析
│  Analysis            │  → 生成可能的切断位点
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  LLM 规则约束过滤     │  考虑：手性控制 / 保护基策略
│                      │  / 原子经济性 / 安全性 / 成本
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  Multi-path Ranking  │  按综合评分排序（绿色化学指数
│                      │  + 可行性 + 成本 + 时间）
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  输出结构化方案        │  → 完整实验步骤 + 参考文献
└──────────────────────┘
```

**技术亮点**：
- 融合 RDKit 的 BRICS 逆合成断裂规则与 LLM 的化学反应机理知识
- 支持约束条件注入（"无重金属催化"、"温度≤80°C"、"三步以内"）
- 每次推荐附带化学原理说明（如"选用BOC保护基的原因是..."）

### 3. CompChem Interpreter —— 计算化学后处理分析模块

**输入格式支持**：
- Materials Studio: `.xsd`, `.castep`, `.xcd`, `.txt` (exported data)
- VASP: `OUTCAR`, `DOSCAR`, `EIGENVAL`
- Gaussian: `.log`, `.out`, `.fchk`
- CP2K: `output`, `ener‑xyz` trajectory

**分析能力**：

| 分析类型 | 输入 | 输出 |
|---------|------|------|
| 能带结构 | 能带数据 (k-point vs energy) | 带隙/带隙类型/有效质量/轨道成分 |
| 态密度 | DOS 数据 (energy vs intensity) | 分波态密度分析/杂化程度判定 |
| 电荷分析 | Mulliken/Hirshfeld 电荷 | 电荷转移量/氧化态推断 |
| 过渡态 | .log 文件 | 活化能/Eyring方程参数 |
| 分子动力学 | 轨迹文件 | RMSD/均方位移/径向分布函数 |
| 光学性质 | 介电函数数据 | 吸收光谱/折射率/反射率 |

**输出示例**（能带分析自动报告）：

```
═══ 能带结构分析报告 ═══

📋 基础信息
   材料：TiO₂ Anatase (101)
   方法：PBE0 (DFT+U)
   截断能：500 eV

📊 能带参数
   • 带隙类型：间接带隙（价带顶：M点，导带底：Γ点）
   • 带隙宽度：3.20 eV（与文献值 3.2-3.4 eV 吻合）
   • 价带顶有效质量：m*_h = 0.82 m₀
   • 导带底有效质量：m*_e = 0.28 m₀

🔬 物理化学意义
   间接带隙意味着电子-空穴复合速率相对较慢，有利于光催化反应中
   载流子的分离与迁移。3.2 eV的带隙对应的吸收边约为387 nm，处于
   紫外区——这是TiO₂在可见光下催化活性有限的主要原因。

💡 优化建议
   可考虑N掺杂（替换部分O原子）或引入氧空位，预期可将带隙缩小
   至2.0-2.8 eV范围，拓展光响应至可见光区。
```

### 4. AutoLab —— 实验数据到文档的端到端管线

**工作流**：
```
原始数据 (CSV/Excel/.txt)
    │
    ▼
┌────────────────┐
│  数据清洗与校验  │  异常值检测 / 单位统一 / 缺失值处理
└───────┬────────┘
        ▼
┌────────────────┐
│  统计分析      │  平均值 / SD / RSD / t检验 / ANOVA
└───────┬────────┘
        ▼
┌────────────────┐
│  数据可视化     │  matplotlib/plotly自动图表
└───────┬────────┘
        ▼
┌────────────────┐
│  报告生成      │  Word (python-docx) / LaTeX / Markdown
└────────────────┘
```

**模板支持**：
- 标准实验报告（含目的/原理/步骤/数据/讨论/结论）
- 学期论文 / 课程设计
- 组会PPT提纲
- 投稿信（Cover Letter）草稿

## MiMo Token 的用途规划

| 用途 | 预估Token消耗 | 说明 |
|------|-------------|------|
| RAG知识库构建（语料嵌入） | ~5亿 | 5000篇文献的向量化嵌入 |
| 文献解析与批量处理 | ~30亿 | 日常文献调研 + 结构化提取 |
| Agent多轮推理 | ~20亿 | 合成路线设计 + 多步推理 |
| 知识蒸馏与模型微调数据准备 | ~10亿 | 构建化学领域专用指令数据集 |
| 持续迭代测试 | ~15亿 | 提示词优化与能力评估 |

**为什么选择 MiMo V2.5：**
1. **超长上下文（1M token）**—— 化学文献+SI往往50-80页，当前许多模型只能分段处理，MiMo的长窗口可以实现"整篇论文一次读完"，文献理解质量有质的提升
2. **MIT开源协议**—— 可在学术场景自由部署与二次开发
3. **算力生态支持**—— Orbit计划的Token Plan可支撑知识库构建阶段的批量调用需求
